pandas + matplotlib — Reference Card

Series/DataFrame · selection · groupby · pivot/melt · merge · time-series · IO · dtypes · matplotlib. Stack: pandas ≥2.2 · matplotlib ≥3.8 · Python ≥3.10 · Modern: Copy-on-Write, PyArrow backend, method chaining

◆ Master pattern (every pandas script)

  1. Load — read_parquet/csv/sql; declare dtypes explicitly
  2. Enable CoW — pd.options.mode.copy_on_write = True (default in pd 3.x)
  3. Inspect — .info() .describe() .head() .isna().sum()
  4. Clean — dtypes, index, missing, duplicates
  5. Transform — method chain: .assign().query().groupby().agg()
  6. Persist — to_parquet (columnar, typed, fast) > CSV
Golden rule (pd 3.x): chained assignment (df["a"][mask] = v) FAILS silently. Always use df.loc[mask, "a"] = v.

▲ groupby + agg (the workhorse)

Single + multi aggregations
g = df.groupby("region") g["gross"].sum() g["gross"].agg(["sum","mean","std","count"]) g.agg({"gross":"sum", "qty":"mean"}) # different per column g.agg(total=("gross","sum"), avg=("gross","mean")) # named agg (recommended)
Multi-key + observed=
df.groupby(["region","segment"], observed=True).size() # observed=True IS RECOMMENDED for categorical keys — pandas 3.x default
Transform (aligned back)
df["z_by_region"] = (df["x"] - df.groupby("region")["x"].transform("mean") ) / df.groupby("region")["x"].transform("std") # transform returns a Series with the ORIGINAL index — perfect for feature eng
Apply (last resort)
df.groupby("region").apply(lambda g: g.nlargest(3, "gross"), include_groups=False) # include_groups=False is the pd ≥2.2 default — silences the DeprecationWarning

▲ Method chaining (readable pipelines)

out = (df .rename(columns={"amt":"amount"}) .query("region in ['EU','US'] and status == 'settled'") .assign(month=lambda d: d["dt"].dt.to_period("M"), gross=lambda d: d["amount"] * d["qty"]) .groupby(["month","region"], observed=True) .agg(total=("gross","sum"), n=("gross","size")) .reset_index() .sort_values(["month","total"], ascending=[True, False]) )
Use .assign(x=lambda d: …) so the LAMBDA sees the CURRENT chain state. Direct expressions read the ORIGINAL df.

def peek(d, label): print(label, d.shape); return d out = (df.query("...") .pipe(peek, "after query") .groupby("g").sum() .pipe(peek, "after agg"))

◆ Performance patterns

  • NEVER iterate rows — vectorised ops are 100-1000× faster
  • Category dtype — 10× memory reduction + faster groupby on strings
  • Filter early — df.query() before any groupby
  • Column selection — read_parquet(columns=[…]) for column pruning
  • eval / query for large frames — numexpr backend, avoids intermediates
  • PyArrow backend — dtype_backend="pyarrow"; faster strings, native nullable
  • Sort once; keep sorted (index-based ops assume sorted)
  • Chunked read — read_csv(chunksize=100_000) or read_sql(chunksize=)

▲ Top traps (6-month memory)

  • Chained assignment df["a"][mask] = v — silently no-ops in pd 3.x (CoW). Use df.loc[mask, "a"] = v.
  • SettingWithCopyWarning — was pd <2.x; CoW eliminated it. If you see it, you're on legacy pandas.
  • inplace=True — inconsistent, disables chaining, no perf win. Prefer reassignment.
  • df.append() — REMOVED in pd 2.x. Use pd.concat([df, row]).
  • pd.get_dummies in prod — train-vs-serve column drift. Use sklearn OneHotEncoder.
  • Datetime freq "M" — deprecated in pd ≥2.2. Use "ME" (month-end) or "MS" (month-start).
  • Hour "H" — deprecated → lowercase "h". Same for "T"→"min", "S"→"s".
  • fillna(method="ffill") — deprecated. Use df.ffill() / df.bfill().
  • Rolling leakage — df["y"].rolling(7).mean() includes CURRENT row. For features: .shift(1).rolling(7).mean().
  • Groupby on category without observed=True — creates rows for unused combinations; huge memory blow-up.
  • Merging on unsorted DatetimeIndex for merge_asof — silently wrong. Sort both sides first.
  • Reading CSV without dtype — pandas infers; dates become object, ints become float on any NaN.
  • df.values in new code — use .to_numpy(); .values is legacy + type-unsafe.
  • Timezone naïveté — mixing naïve + tz-aware raises. Localize on read.
  • Copy vs view uncertainty — df.copy() if you need a guaranteed independent frame.
