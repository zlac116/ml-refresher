PostgreSQL — Reference Card

psql · DDL · DML · SELECT · JOINs · CTE · window · JSON · UPSERT · indexes · transactions · plans. Stack: PostgreSQL ≥15 (MERGE, IDENTITY, generated cols) · SQL:2023 · Modern: `RETURNING`, `ON CONFLICT`, `LATERAL`, `FILTER`, materialised CTEs

◆ Query anatomy (order OF EXECUTION, not writing)

-- WRITE order -- EXECUTION order (know both) SELECT ... -- 5. SELECT projections FROM t -- 1. FROM (+ JOINs) JOIN u ON ... -- 2. ON / JOIN filters WHERE ... -- 3. WHERE GROUP BY ... -- 4. GROUP BY + aggregates HAVING ... -- 6. HAVING ORDER BY ... -- 7. ORDER BY LIMIT n OFFSET m; -- 8. LIMIT / OFFSET
Alias order: aliases from SELECT are NOT visible in WHERE/GROUP BY (they run before SELECT). But they ARE visible in ORDER BY (runs after). Use a CTE or subquery to sidestep.

◆ Starter · minimal CRUD skeleton

-- DDL (modern PG: IDENTITY, GENERATED, CHECK) CREATE TABLE trades ( id bigint GENERATED ALWAYS AS IDENTITY PRIMARY KEY, symbol text NOT NULL CHECK (symbol ~ '^[A-Z]+$'), qty numeric(18,4) NOT NULL, price numeric(18,4) NOT NULL, gross numeric(18,4) GENERATED ALWAYS AS (qty * price) STORED, ts timestamptz NOT NULL DEFAULT now(), tags jsonb NOT NULL DEFAULT '{}'::jsonb ); CREATE INDEX ON trades (symbol, ts DESC); -- covering + sort CREATE INDEX ON trades USING gin (tags); -- JSONB search -- DML (always use RETURNING; batch when possible) INSERT INTO trades (symbol, qty, price) VALUES ('AAPL', 100, 180.5), ('MSFT', 50, 310.0) RETURNING id, gross; -- SELECT SELECT symbol, sum(gross) FILTER (WHERE qty > 0) AS long_gross FROM trades WHERE ts >= now() - interval '7 days' GROUP BY symbol HAVING sum(gross) > 1000 ORDER BY long_gross DESC LIMIT 10; -- UPDATE + DELETE with RETURNING (audit) UPDATE trades SET price = 181.0 WHERE id = 42 RETURNING *; DELETE FROM trades WHERE ts < now() - interval '1 year' RETURNING id;

● DDL — define schema (modern)

CREATE TABLE users ( id bigint GENERATED ALWAYS AS IDENTITY PRIMARY KEY, -- (not SERIAL — legacy) email citext NOT NULL UNIQUE, -- case-insensitive text created timestamptz NOT NULL DEFAULT now(), CONSTRAINT email_looks_ok CHECK (email ~ '@')); -- Generated columns (STORED = computed on write, indexable) ALTER TABLE trades ADD COLUMN gross numeric GENERATED ALWAYS AS (qty*price) STORED; -- Foreign keys with cascade CREATE TABLE orders ( id bigint GENERATED ALWAYS AS IDENTITY PRIMARY KEY, user_id bigint NOT NULL REFERENCES users(id) ON DELETE CASCADE); -- Enum types CREATE TYPE status AS ENUM ('open', 'filled', 'cancelled'); -- Partitioning (large tables) CREATE TABLE events (id bigint, dt timestamptz, ...) PARTITION BY RANGE (dt); CREATE TABLE events_2026 PARTITION OF events FOR VALUES FROM ('2026-01-01') TO ('2027-01-01'); -- Drop / alter ALTER TABLE trades RENAME COLUMN qty TO quantity; ALTER TABLE trades ADD COLUMN venue text; DROP TABLE IF EXISTS temp_data CASCADE;
Use IDENTITY not SERIAL (SQL standard, safer around insert-explicit-id). timestamptz not timestamp (stores UTC internally).

● SELECT essentials

SELECT DISTINCT ON (symbol) symbol, price, ts -- latest row per symbol FROM trades ORDER BY symbol, ts DESC; SELECT * FROM trades WHERE ts BETWEEN '2026-01-01' AND '2026-12-31' AND symbol IN ('AAPL','MSFT','GOOG') AND tags ->> 'strategy' = 'momentum' AND qty IS NOT NULL; -- Boolean / conditional SELECT symbol, CASE WHEN qty > 0 THEN 'long' WHEN qty < 0 THEN 'short' ELSE 'flat' END AS side, COALESCE(tags->>'venue', 'unknown') AS venue, NULLIF(gross, 0) AS gross_nz -- returns NULL if gross=0 FROM trades; -- Existence / negation (prefer over IN for large sets) SELECT * FROM trades t WHERE EXISTS (SELECT 1 FROM users u WHERE u.id=t.user_id); SELECT * FROM trades t WHERE NOT EXISTS (SELECT 1 FROM refunds r WHERE r.tid=t.id);

SELECT o.id, u.email, u.name FROM orders o JOIN users u ON u.id = o.user_id -- users.id + orders.user_id (see DDL above) LEFT JOIN discounts d ON d.user_id = u.id AND d.active WHERE o.status = 'open'; -- USING (col) only works when BOTH tables have SAME-NAMED column -- LATERAL — top-N per group (elegant) SELECT u.id, top3.* FROM users u CROSS JOIN LATERAL ( SELECT o.id, o.amount FROM orders o WHERE o.user_id = u.id ORDER BY o.amount DESC LIMIT 3 ) top3;
Cartesian trap: missing ON clause → CROSS JOIN silently. Turn on set client_min_messages='warning'; to catch stealth cross-products.

▲ Aggregates + GROUP BY + FILTER

SELECT symbol, count(*) AS n, count(*) FILTER (WHERE qty > 0) AS long_n, sum(gross) AS total, sum(gross) FILTER (WHERE qty > 0) AS long_gross, avg(price), percentile_cont(0.5) WITHIN GROUP (ORDER BY price) AS median, string_agg(DISTINCT venue, ', ' ORDER BY venue) AS venues, array_agg(id ORDER BY ts) AS ids FROM trades WHERE ts >= now() - interval '30 days' GROUP BY symbol HAVING count(*) > 5 ORDER BY total DESC; -- Subtotals: GROUPING SETS / ROLLUP / CUBE SELECT region, product, sum(gross) FROM sales GROUP BY ROLLUP (region, product); -- (region,product) + (region) + total
FILTER (WHERE ...) replaces sum(CASE WHEN ...) — cleaner + slightly faster.

◆ Window functions (aggregate without collapsing)

SELECT symbol, ts, price, -- Ranking row_number() OVER w AS rn, -- 1,2,3 (no ties) rank() OVER w AS rk, -- 1,2,2,4 (gaps) dense_rank() OVER w AS drk, -- 1,2,2,3 (no gaps) percent_rank() OVER w AS pct_rk, ntile(4) OVER w AS quartile, -- Aggregate (over frame) sum(price) OVER w AS running_sum, avg(price) OVER (PARTITION BY symbol ORDER BY ts ROWS BETWEEN 6 PRECEDING AND CURRENT ROW) AS ma7, -- Neighbours lag(price,1) OVER w AS prev_price, lead(price,1) OVER w AS next_price, first_value(price) OVER w AS first_px, last_value(price) OVER (PARTITION BY symbol ORDER BY ts ROWS BETWEEN UNBOUNDED PRECEDING AND UNBOUNDED FOLLOWING) AS last_px FROM trades WINDOW w AS (PARTITION BY symbol ORDER BY ts); -- WINDOW named clause — DRY when reusing the same partition/order many times
Frame clauseMeaning
ROWS N PRECEDINGfixed count of rows
RANGE N PRECEDINGvalues within N of current (needs numeric/date ORDER)
GROUPS N PRECEDINGPG 11+; per ORDER BY tie group
UNBOUNDED PRECEDINGfrom start of partition
UNBOUNDED FOLLOWINGto end of partition
Default frame when using ORDER BY without explicit frame = RANGE UNBOUNDED PRECEDING (running total by value-of-ORDER-BY). Silently wrong for last_value() — specify frame explicitly.

◆ CTE (WITH) + materialization hints

-- Simple CTE (inlined by optimizer in PG 12+ — single-reference, non-recursive, no side-effects) WITH recent AS ( SELECT * FROM trades WHERE ts >= now() - interval '7 days' ) SELECT symbol, sum(gross) FROM recent GROUP BY symbol; -- Force materialization (optimizer barrier — used when CTE is expensive OR affects semantics) WITH heavy AS MATERIALIZED ( -- PG 12+ SELECT * FROM slow_view ) SELECT * FROM heavy WHERE ...; WITH cheap AS NOT MATERIALIZED ( -- allow inlining even if referenced multiple times SELECT symbol FROM trades WHERE ts > now() - interval '1h' ) SELECT count(*) FROM cheap; -- Recursive CTE (hierarchies, path enumeration) WITH RECURSIVE ancestors AS ( SELECT id, parent_id, 1 AS depth FROM tree WHERE id = 42 -- base case UNION ALL SELECT t.id, t.parent_id, a.depth + 1 -- step case FROM tree t JOIN ancestors a ON t.id = a.parent_id ) SELECT * FROM ancestors; -- Data-modifying CTEs (multi-statement in one round-trip) WITH deleted AS (DELETE FROM staging RETURNING *) INSERT INTO archive SELECT * FROM deleted;

▲ UPSERT — ON CONFLICT vs MERGE

-- ON CONFLICT (idiomatic PG since 9.5 — simple, fast) INSERT INTO trades (id, symbol, qty) VALUES (1, 'AAPL', 100) ON CONFLICT (id) DO UPDATE SET qty = EXCLUDED.qty, -- EXCLUDED = the row we tried to insert updated_at = now() WHERE trades.qty IS DISTINCT FROM EXCLUDED.qty -- skip no-op updates RETURNING id; -- MERGE (PG 15+ — SQL standard, better for complex cases) MERGE INTO trades t USING (VALUES (1, 'AAPL', 100), (2, 'MSFT', 50)) AS s(id, symbol, qty) ON t.id = s.id WHEN MATCHED AND s.qty IS NULL THEN DELETE -- ← ORDER matters: first matching branch wins WHEN MATCHED AND t.qty IS DISTINCT FROM s.qty THEN UPDATE SET qty = s.qty, updated_at = now() WHEN NOT MATCHED THEN INSERT (id, symbol, qty) VALUES (s.id, s.symbol, s.qty); -- PG 17+: MERGE ... RETURNING supported. -- Also common: INSERT ... ON CONFLICT (pk) DO NOTHING (idempotent-insert)
ON CONFLICTMERGE

-- Production-safe (never blocks writes; slower) CREATE INDEX CONCURRENTLY idx_trades_symbol_ts ON trades (symbol, ts DESC) INCLUDE (gross) -- covering — avoid heap fetch WHERE ts >= '2026-01-01'; -- partial — smaller + faster CREATE UNIQUE INDEX ON users (lower(email)); -- functional (case-insens login) -- Rebuild without lock (PG 12+) REINDEX INDEX CONCURRENTLY idx_trades_symbol_ts; -- Check unused / bloat SELECT indexname, idx_scan FROM pg_stat_user_indexes ORDER BY idx_scan; -- 0 = unused
Never CREATE INDEX without CONCURRENTLY on a live table — blocks writes for the entire build.

◆ Maintenance (the things that bite in prod)

  • VACUUM / autovacuum — reclaims dead tuples. Watch bloat: SELECT * FROM pg_stat_user_tables ORDER BY n_dead_tup DESC;
  • ANALYZE — refreshes stats for the planner. Runs with VACUUM (autovacuum default).
  • REINDEX CONCURRENTLY — rebuilds bloated index without blocking.
  • pg_stat_activity — see live queries: SELECT pid, state, query FROM pg_stat_activity;
  • pg_stat_statements — top queries by total time (enable in postgresql.conf).
  • Long-running txn: SELECT pid, xact_start FROM pg_stat_activity WHERE xact_start < now()-interval '5 min';
  • Kill query: SELECT pg_cancel_backend(pid); (soft) or pg_terminate_backend(pid) (kill).
  • Backups: pg_dump -Fc db > db.dump then pg_restore -d newdb db.dump.
  • Migrations: Alembic / Flyway / sqitch. Never migrate schema without a tested rollback.

▲ Top traps (6-month memory)

  • NULL ≠ NULL — WHERE x = NULL matches nothing; use x IS NULL.
  • NULL + anything = NULL — arithmetic silently returns NULL. Use COALESCE(x, 0).
  • count(*) vs count(col) — count(col) skips NULLs; count(*) counts rows.
  • Chained aggregate not allowed — sum(avg(x)) raises. Wrap inner in a CTE.
  • DISTINCT ON needs a matching leading ORDER BY. Missing = non-determinism.
  • SERIAL is legacy — use GENERATED ALWAYS AS IDENTITY.
  • timestamp without tz — silently drops tz info. Use timestamptz.
  • CREATE INDEX without CONCURRENTLY blocks writes.
  • Function calls in WHERE (WHERE lower(email)=...) skip indexes unless there's a functional index.
  • LIKE 'foo%' uses btree; LIKE '%foo' or '%foo%' does NOT — use pg_trgm GIN.
  • OR in WHERE often defeats indexes. Rewrite as UNION ALL.
  • Window frame default is RANGE UNBOUNDED PRECEDING — silently wrong for last_value.
  • UPDATE without WHERE — hits every row. Always test in a transaction first.
  • Long-running IDLE IN TRANSACTION — blocks vacuum. Set idle_in_transaction_session_timeout.
  • MERGE not concurrent-safe — needs SERIALIZABLE + retry. ON CONFLICT is.
