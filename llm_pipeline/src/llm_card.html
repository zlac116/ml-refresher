LLM Pipeline (LangChain + LangGraph) — Project Recipe Card

2 paradigms · 10-line agent · LCEL · StateGraph · tools · RAG · multi-agent · HITL · streaming · observability. Deep companion: llm_cheatsheet.md + llm_pipeline.ipynb

◆ The 2 paradigms (decide FIRST)

Work shapeTool
Linear: input → transform → model → parseLCEL (| pipe)
RAG (retrieve → augment → generate)LCEL
Few-shot · summarise · structured extractLCEL
Tool-calling agent (loop until done)LangGraph
Multi-agent · routingLangGraph
HITL approval gatesLangGraph
Branching/long-running conversationLangGraph
Rule: linear = LCEL. Conditional / iterative = LangGraph. Don't shoehorn a state machine into LCEL; don't wrap a 3-line RAG chain in a graph.

◆ LCEL pattern (4 steps, always the same)

chain = prompt | model | parser # 1. COMPOSE: pipe Runnables chain.invoke({"input": ...}) # 2. INVOKE: sync, single shot async for chunk in chain.astream({...}): ... # 3. STREAM: async tokens chain.batch([{...}, {...}, {...}]) # 4. BATCH: concurrent runs # Every Runnable supports invoke / ainvoke / stream / astream / batch / abatch

◆ LangGraph pattern (5 steps, always the same)

from typing import Annotated, TypedDict from langgraph.graph import StateGraph, START, END from langgraph.graph.message import add_messages from langgraph.checkpoint.memory import InMemorySaver # 1. STATE — shape that flows between nodes (TypedDict + reducers) class State(TypedDict): messages: Annotated[list, add_messages] # 2. NODES — pure functions state -> state-update (dict) def my_node(state: State) -> dict: return {"messages": [{"role": "assistant", "content": "..."}]} # 3. GRAPH — add nodes + edges + START/END builder = StateGraph(State) builder.add_node("step1", my_node) builder.add_edge(START, "step1") builder.add_edge("step1", END) # 4. COMPILE with checkpointer (for memory + HITL) graph = builder.compile(checkpointer=InMemorySaver()) # 5. RUN with thread_id for persistence config = {"configurable": {"thread_id": "user-42"}} out = graph.invoke({"messages": [{"role": "user", "content": "hi"}]}, config)
Everything is a Runnable. Both graph and chain implement the same interface (invoke/stream/batch).

◆ Starter · 10-line tool-calling agent (LangChain 1.x)

# Canonical create_agent (LangChain ≥1.3 + LangGraph ≥1.2) from langchain.agents import create_agent from langchain.tools import tool @tool def get_weather(city: str) -> str: """Get weather for a given city.""" return f"It's sunny in {city}" agent = create_agent( model="anthropic:claude-sonnet-4-6", # provider:model string tools=[get_weather], system_prompt="You are a helpful assistant.", ) out = agent.invoke({"messages": [{"role": "user", "content": "weather in Paris?"}]}) print(out["messages"][-1].content)
API change: langgraph.prebuilt.create_react_agent (legacy) → langchain.agents.create_agent (1.x). prompt= kwarg → system_prompt=.

▲ Tools (function calling)

@tool decorator (canonical)
from langchain.tools import tool from pydantic import BaseModel, Field import yfinance as yf @tool def get_price(symbol: str) -> float: """Return latest close price for the given ticker (USD).""" return float(yf.Ticker(symbol).history(period="1d")["Close"].iloc[-1]) # Structured args via Pydantic for richer schemas: class SearchInput(BaseModel): query: str = Field(..., description="search query") k: int = Field(default=5, ge=1, le=20) @tool(args_schema=SearchInput) def search(query: str, k: int = 5) -> list[str]: """Search the knowledge base.""" return retriever.invoke(query)[:k]
Docstring becomes the tool description sent to the model — write it for the model.
Bind to model directly (no agent)
m_with_tools = m.bind_tools([get_price, search]) out = m_with_tools.invoke("price of AAPL?") # out.tool_calls == [{"name":"get_price","args":{"symbol":"AAPL"},"id":"..."}] for tc in out.tool_calls: result = {"get_price": get_price, "search": search}[tc["name"]].invoke(tc["args"]) # Feed result back as ToolMessage with matching id...
Error handling
from langchain_core.tools import ToolException @tool def risky(): """Do something that might fail.""" raise ToolException("retryable error msg") # Agent will receive the exception message as a ToolMessage and can recover.

▲ Agents — create_agent (1.x)

Canonical agent
from langchain.agents import create_agent from langgraph.checkpoint.memory import InMemorySaver agent = create_agent( model="anthropic:claude-sonnet-4-6", # or an instantiated model tools=[get_price, search], system_prompt="You answer using the supplied tools.", checkpointer=InMemorySaver(), # thread-scoped memory ) config = {"configurable": {"thread_id": "user-42"}} agent.invoke({"messages": [{"role":"user","content":"q?"}]}, config)
Old API: create_react_agent(model, tools, prompt=...) → New API: create_agent(model, tools, system_prompt=...). The deprecated wrapper still works as of 1.3 but emits warnings.
Customising (1.x kwargs)
  • response_format=PydanticModel — typed final answer (read via out["structured_response"], NOT messages)
  • state_schema=MyState — extend the default messages-only state
  • middleware=[...] — pre/post/around model hooks (1.x replacement for legacy hooks). See Middleware section.
  • checkpointer=InMemorySaver() — thread-scoped memory + HITL prerequisite
API change in 1.x: interrupt_before=, pre_model_hook=, post_model_hook= are GONE. HITL + hooks are now configured via middleware=[...].
Stream agent steps
for ev in agent.stream({"messages": [...]}, config, stream_mode="updates"): print(ev) # node-by-node state updates # stream_mode: "values" (full state) / "messages" (tokens) / "updates" (deltas)

▲ Middleware (the 1.x extensibility primitive)

from langchain.agents import create_agent from langchain.agents.middleware import ( HumanInTheLoopMiddleware, # HITL approval gates per tool SummarizationMiddleware, # context-window management ModelFallbackMiddleware, # auto-fallback to cheaper model # plus AnthropicPromptCachingMiddleware, PIIRedactionMiddleware, etc. ) agent = create_agent( model="anthropic:claude-sonnet-4-6", tools=[get_price, search], system_prompt="You are a trading assistant.", middleware=[ HumanInTheLoopMiddleware(interrupt_on={"get_price": True}), # approve only get_price SummarizationMiddleware(model="openai:gpt-4o-mini", max_tokens=4000), ModelFallbackMiddleware(fallback_model="openai:gpt-4o-mini"), ], checkpointer=InMemorySaver(), )
Use middleware for: HITL gates, context summarisation, model fallback, retries, prompt caching, PII redaction, dynamic prompts, dynamic tools. Compose multiple — order matters (outermost first).

▲ ToolNode (canonical tool executor for graphs)

from langgraph.prebuilt import ToolNode, tools_condition tools = [get_price, search] tool_node = ToolNode(tools) # parallel calls + error handling + ToolMessage shaping m_with_tools = m.bind_tools(tools) def call_model(state): return {"messages": [m_with_tools.invoke(state["messages"])]} builder = StateGraph(MessagesState) builder.add_node("agent", call_model) builder.add_node("tools", tool_node) builder.add_edge(START, "agent") builder.add_conditional_edges("agent", tools_condition) # built-in router builder.add_edge("tools", "agent") graph = builder.compile()
Use ToolNode in custom graphs; create_agent wires it for you.

● LangGraph StateGraph (custom)

Built-in reducer
from langgraph.graph import MessagesState # MessagesState = TypedDict with messages: Annotated[list, add_messages] # Use directly if your state is just messages.
Custom state with reducers
from typing import Annotated, TypedDict import operator class State(TypedDict): messages: Annotated[list, add_messages] # append + dedupe by ID scratch: Annotated[list[str], operator.add] # plain concat count: int # plain overwrite
Annotated[T, reducer] tells LangGraph how to merge concurrent updates.
Conditional edges
def router(state: State) -> str: return "yes" if needs_tool(state) else "no" builder.add_conditional_edges( "agent", router, {"yes": "tools", "no": END}) # Or: return list of next nodes (parallel fan-out) def fan_out(state) -> list[str]: return ["a", "b"] builder.add_conditional_edges("split", fan_out, ["a", "b"])

▲ Multi-agent (Command + supervisor)

Command (state-update + routing)
from langgraph.types import Command from typing import Literal def supervisor(state) -> Command[Literal["researcher","writer","__end__"]]: decision = decide_next(state) return Command( update={"messages": [...]}, # state update (optional) goto=decision, # next node — overrides static edges ) # Use in node return; with conditional edges; or to resume an interrupt # (resume= is INPUT-only to invoke/stream — don't conflate with goto/update).
Supervisor pattern
researcher = create_agent(model, tools=[search], system_prompt="research") writer = create_agent(model, tools=[], system_prompt="write") class State(MessagesState): next: str def supervisor(state): # LLM decides next worker return Command(goto="researcher" if needs_data(state) else "writer") builder = StateGraph(State) builder.add_node("supervisor", supervisor) builder.add_node("researcher", researcher) builder.add_node("writer", writer) builder.add_edge(START, "supervisor") builder.add_edge("researcher", "supervisor") # loop back to supervisor builder.add_edge("writer", END) graph = builder.compile(checkpointer=InMemorySaver())

◆ HITL — interrupt + resume

from langgraph.types import interrupt, Command def review(state): decision = interrupt({"propose": state["draft"]}) # pauses execution return {"messages": [{"role":"user","content": decision}]} config = {"configurable": {"thread_id": "1"}} # Run hits interrupt; returned state contains __interrupt__: out = graph.invoke({"draft": "..."}, config) # Resume with user's answer: graph.invoke(Command(resume="approved"), config)
Requires checkpointer. Interrupt persists state to the checkpointer; resume hydrates it. No checkpointer = no HITL.
Don't confuse: Command(resume=...) = INPUT to invoke/stream. Command(update=..., goto=...) = RETURN from a node. They are different roles of the same class.

◆ RAG patterns

Basic RAG (LCEL)
from langchain_chroma import Chroma from langchain_openai import OpenAIEmbeddings from langchain_text_splitters import RecursiveCharacterTextSplitter from langchain_core.runnables import RunnablePassthrough from langchain_core.output_parsers import StrOutputParser def format_docs(docs): # define before use! return "\n\n".join(d.page_content for d in docs) splits = RecursiveCharacterTextSplitter(chunk_size=800, chunk_overlap=100 ).split_documents(docs) vs = Chroma.from_documents(splits, OpenAIEmbeddings()) retriever = vs.as_retriever(search_kwargs={"k": 4}) rag = ( {"context": retriever | format_docs, "question": RunnablePassthrough()} | prompt | model | StrOutputParser() ) rag.invoke("How does X work?")

◆ Deployment (FastAPI)

from fastapi import FastAPI from fastapi.responses import StreamingResponse from pydantic import BaseModel from langgraph.checkpoint.postgres import AsyncPostgresSaver import json app = FastAPI() class Req(BaseModel): q: str thread_id: str # Async-first: agent.ainvoke / agent.astream do NOT block the event loop. @app.post("/invoke") async def invoke(r: Req): cfg = {"configurable": {"thread_id": r.thread_id}} out = await agent.ainvoke({"messages": [{"role":"user","content": r.q}]}, cfg) return {"answer": out["messages"][-1].content} @app.post("/stream") async def stream(r: Req): cfg = {"configurable": {"thread_id": r.thread_id}} async def gen(): async for ev in agent.astream( {"messages":[{"role":"user","content": r.q}]}, cfg, stream_mode="messages"): chunk = ev[0] yield f"data: {json.dumps({'chunk': chunk.content})}\n\n" return StreamingResponse(gen(), media_type="text/event-stream")
Production checklist: async endpoints + ainvoke/astream; load agent at FastAPI lifespan (not import-time); use AsyncPostgresSaver for cross-process checkpointing; set recursion_limit + per-request timeouts; auth BEFORE trusting thread_id.

▲ Top traps (6-month memory)

  • create_react_agent is legacy. Use langchain.agents.create_agent; prompt= → system_prompt=.
  • MemorySaver renamed → InMemorySaver. Old name deprecated.
  • set_entry_point/set_finish_point legacy. Use add_edge(START, ...) / add_edge(..., END).
  • Command roles: resume= is INPUT to invoke; update= + goto= are RETURN from a node. Don't conflate.
  • No checkpointer = no HITL, no memory. Add one at compile time.
  • Tools without docstrings → model can't see what they do; treat docstring as the model's prompt.
  • Streaming mode mismatch: "messages" = tokens; "updates" = node deltas; "values" = full state. Pick deliberately.
  • Forgetting thread_id → fresh state every call; memory invisible.
  • State schema: prefer TypedDict; use Pydantic only when you need validation at node boundaries.
  • response_format= result lives at out["structured_response"], NOT in out["messages"][-1]. Easy silent miss.
  • llm_output is provider-specific (often None); prefer msg.usage_metadata for token counts.
  • Sync FastAPI def with agent.invoke blocks the event loop. Use async def + await agent.ainvoke(...).
  • Caching with structured output: cache key is the prompt — same prompt + different schema returns stale.
  • No tracing → debugging is guessing. Set LANGSMITH_TRACING=true as table-stakes.
  • Recursion limit defaults to 25 — agent loops can silently truncate. Raise via config["recursion_limit"].
