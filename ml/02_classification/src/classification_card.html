Classification — Project Recipe Card

End-to-end recipe · model toolbox · metrics · CV · class imbalance · threshold tuning · calibration · deployment. Deep companion: classification_cheatsheet.md + classification.ipynb

◆ Project recipe (12 steps)

  1. Frame — binary / multi-class / multi-label; success metric tied to business action
  2. Stratified split FIRST — train/val/test before EDA (no leakage)
  3. EDA + label audit — class balance, label noise, leakage candidates
  4. Feature pipeline — ColumnTransformer → impute → encode → scale
  5. Baseline — DummyClassifier (most_frequent) + LogisticRegression
  6. Model toolbox — LR → RF → GBM, climb the ladder
  7. CV with StratifiedKFold — preserve class ratio in each fold
  8. Tune — Optuna TPE (≥3 hp) or GridSearchCV
  9. Calibrate — CalibratedClassifierCV if you need P(y=1) to be a true probability
  10. Threshold — tune on VAL using precision-recall curve (NEVER on test)
  11. Test ONCE — final eval; report confusion matrix + main metric
  12. Deploy — pickle pipeline + log threshold + drift monitor
Universal skeleton: Pipeline([("prep", ColumnTransformer(...)), ("model", LogisticRegression(class_weight="balanced"))]) → tune → calibrate → threshold → ship.

● Frame + label audit

Frame checklist
  • Task: binary / multi-class (k mutually exclusive) / multi-label (k independent)
  • Output: hard label OR probability OR ranking
  • Cost asymmetry: FP cost vs FN cost (drives threshold)
  • Class balance: balanced (acc OK) vs imbalanced (PR-AUC / F1)
  • Label source: hand-labeled / programmatic / proxy?
Label audit

Leakage candidates

Cost-sensitive learning (asymmetric FP/FN)
from sklearn.metrics import make_scorer COST = {"FP": 1.0, "FN": 5.0} # FN 5× costlier (fraud, disease, churn) def custom_cost(y_true, y_pred): tn = ((y_pred==0)&(y_true==0)).sum(); fp = ((y_pred==1)&(y_true==0)).sum() fn = ((y_pred==0)&(y_true==1)).sum(); tp = ((y_pred==1)&(y_true==1)).sum() return -(COST["FP"]*fp + COST["FN"]*fn) # negative — sklearn maximises scorer = make_scorer(custom_cost, greater_is_better=True) # Use as scoring in GridSearchCV / cross_val_score: gs = GridSearchCV(pipe, grid, cv=skf, scoring=scorer) # Or apply class_weight to bake into loss: LogisticRegression(class_weight={0: 1, 1: 5}) # explicit numeric weights # Or per-sample weights: sample_weight = np.where(y_tr == 1, 5.0, 1.0) pipe.fit(X_tr, y_tr, model__sample_weight=sample_weight)

Confusion matrix vocabulary

True 0TNFP (type-I)

Precision = TP/(TP+FP) "of predicted positives, % correct" Recall = TP/(TP+FN) "of true positives, % caught" F1 = 2·P·R/(P+R) harmonic mean Specificity = TN/(TN+FP) "true negative rate"

MetricWhen

PR-AUC (AvgPrec)imbalanced — preferred over ROC

Metric picker
  • Balanced binary → Accuracy + ROC-AUC
  • Imbalanced binary → PR-AUC + F1 (at chosen threshold)
  • Need probabilities → Brier + Log-loss (calibration!)
  • Asymmetric costs → custom utility = TP·v − FP·c
  • Multi-class → macro-F1 + Cohen's κ + confusion matrix

import optuna def objective(trial): params = { "n_estimators": 2000, "learning_rate": trial.suggest_float("lr", 1e-3, 0.3, log=True), "num_leaves": trial.suggest_int("leaves", 15, 255, log=True), "min_child_samples": trial.suggest_int("mcs", 5, 100), "reg_lambda": trial.suggest_float("rl", 1e-3, 10, log=True), "subsample": trial.suggest_float("sub", 0.5, 1.0), "colsample_bytree": trial.suggest_float("col", 0.5, 1.0), } model = lgb.LGBMClassifier(**params, class_weight="balanced", random_state=42) return cross_val_score(model, X_tr, y_tr, cv=5, scoring="roc_auc", n_jobs=-1).mean() study = optuna.create_study(direction="maximize", sampler=optuna.samplers.TPESampler(seed=42), pruner=optuna.pruners.MedianPruner()) study.optimize(objective, n_trials=100)
Threshold tuning (CRITICAL for imbalanced)
from sklearn.metrics import precision_recall_curve p_va = pipe.predict_proba(X_va)[:, 1] prec, rec, thr = precision_recall_curve(y_va, p_va) f1 = 2 * prec * rec / (prec + rec + 1e-12) best_t = thr[np.argmax(f1[:-1])] print("best threshold =", best_t) y_hat_te = (pipe.predict_proba(X_te)[:, 1] > best_t).astype(int)
Tune threshold on VAL set, NEVER on test. Threshold is a hyperparameter.

◆ Calibration + diagnostics

Why calibrate?
RF / SVM / GBM produce scores in [0,1] that don't equal true P(y=1). If you'll use probabilities (Bayesian decisions, expected value), calibrate.
from sklearn.calibration import CalibratedClassifierCV, calibration_curve # Method "isotonic" (non-parametric, needs >1k samples) or "sigmoid" (Platt) cal = CalibratedClassifierCV(estimator=base_model, method="isotonic", cv=5) cal.fit(X_tr, y_tr) # Diagnose: prob_true, prob_pred = calibration_curve(y_te, p, n_bins=10) plt.plot(prob_pred, prob_true, marker="o"); plt.plot([0,1],[0,1], "k--")

Drift monitoring
  • Input drift: PSI / KS per feature vs train
  • Score drift: histogram of P(y=1) over time
  • Label drift: when labels arrive, monitor rolling base-rate
  • Performance drift: rolling ROC-AUC / PR-AUC
  • Re-train: PSI > 0.25 OR perf drop > 5%

▲ Top traps (6-month memory)

  • Accuracy on imbalanced data — 99% accuracy on 1% positives = "predict 0 always". Use PR-AUC / F1.
  • Tuning threshold on test — threshold is a hyperparameter; tune on VAL.
  • Target encoding outside CV — fits on full data → leakage. Use sklearn 1.3+ TargetEncoder with cv.
  • Scaler fit on all data — fits on test set rows; use Pipeline.
  • SMOTE before splitting — synthetic positives leak into test. Resample INSIDE CV via imblearn.Pipeline.
  • Forgetting calibration — RF/SVM/GBM scores ≠ probabilities. Calibrate if you use them as P(y=1).
  • Train-serve schema mismatch — pin column order + dtypes in deployment bundle.
  • Single-model feature importance — different models disagree. Combine SHAP + permutation.
