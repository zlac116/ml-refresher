Neural Networks (PyTorch) — Project Recipe Card

9-step recipe · training loop · loss/optimizer table · regularisation · debug checklist · persistence. Deep companion: neural_networks_cheatsheet.md + neural_networks.ipynb

◆ 9-step recipe

  1. Setup — seeds, device, deterministic flags
  2. Split — train / val / test (3 sets, NOT 2)
  3. Scale — fit scaler on TRAIN only; transform val/test
  4. Tensors + DataLoader — shuffle TRAIN only; batch size powers of 2
  5. Model — nn.Module subclass or nn.Sequential
  6. Loss + Optimizer — match to task; Adam @ 1e-3 default
  7. Training loop — zero_grad → forward → loss → backward → step
  8. Evaluate on TEST — once, after tuning is done
  9. Diagnose — loss curves, regularise, refit
Universal contract: output layer + dtype + loss MUST agree. Mismatch = silent bug (NaN loss, near-random accuracy, exploding gradients).

◆ Starter skeleton · B: model + train loop

# 5. MODEL (MLP) class MLP(nn.Module): def __init__(self, in_dim, hidden=64, out_dim=1, p=0.2): super().__init__() self.net = nn.Sequential( nn.Linear(in_dim, hidden), nn.ReLU(), nn.Dropout(p), nn.Linear(hidden, hidden), nn.ReLU(), nn.Dropout(p), nn.Linear(hidden, out_dim)) def forward(self, x): return self.net(x) model = MLP(X_tr.shape[1], hidden=64, out_dim=1).to(device) # 6. LOSS + OPTIMIZER (regression example) loss_fn = nn.MSELoss() opt = torch.optim.Adam(model.parameters(), lr=1e-3, weight_decay=1e-4) scheduler = torch.optim.lr_scheduler.ReduceLROnPlateau(opt, patience=5, factor=0.5) # 7. TRAIN LOOP with early stopping best_val, patience, bad = float("inf"), 10, 0 hist = {"tr": [], "va": []} for epoch in range(200): model.train() tr_loss = 0.0 for xb, yb in dl_tr: xb, yb = xb.to(device), yb.to(device) opt.zero_grad() pred = model(xb).squeeze(-1) loss = loss_fn(pred, yb) loss.backward() opt.step() tr_loss += loss.item() * xb.size(0) tr_loss /= len(dl_tr.dataset) model.eval() va_loss = 0.0 with torch.no_grad(): for xb, yb in dl_va: xb, yb = xb.to(device), yb.to(device) pred = model(xb).squeeze(-1) va_loss += loss_fn(pred, yb).item() * xb.size(0) va_loss /= len(dl_va.dataset) scheduler.step(va_loss) hist["tr"].append(tr_loss); hist["va"].append(va_loss) if va_loss < best_val - 1e-4: best_val, bad = va_loss, 0 torch.save(model.state_dict(), "best.pt") else: bad += 1 if bad >= patience: print(f"early stop at epoch {epoch}, best val={best_val:.4f}") break # 8. EVAL on TEST — ONCE model.load_state_dict(torch.load("best.pt")) model.eval() with torch.no_grad(): preds = torch.cat([model(xb.to(device)).cpu() for xb, _ in dl_te]).squeeze(-1).numpy() # Compute metrics in sklearn (RMSE, R², ROC-AUC, F1, ...)

▲ Architecture patterns

Tabular MLP (the default)
class MLP(nn.Module): def __init__(self, in_dim, hidden=(128, 64), out_dim=1, p=0.2, bn=False): super().__init__() layers = [] prev = in_dim for h in hidden: layers += [nn.Linear(prev, h)] if bn: layers += [nn.BatchNorm1d(h)] layers += [nn.ReLU(inplace=True), nn.Dropout(p)] prev = h layers += [nn.Linear(prev, out_dim)] self.net = nn.Sequential(*layers) def forward(self, x): return self.net(x)
2-3 hidden layers, hidden width 64-256, ReLU, Dropout 0.1-0.3, optional BN.
Embeddings for categorical
class TabularNN(nn.Module): def __init__(self, cat_cardinalities, n_cont, hidden=128, out_dim=1): super().__init__() # Embedding dim heuristic: min(50, (k+1)//2) self.embs = nn.ModuleList([ nn.Embedding(k, min(50, (k+1)//2)) for k in cat_cardinalities]) emb_dim = sum(e.embedding_dim for e in self.embs) self.bn = nn.BatchNorm1d(n_cont) self.head = nn.Sequential( nn.Linear(emb_dim + n_cont, hidden), nn.ReLU(), nn.Dropout(0.2), nn.Linear(hidden, out_dim)) def forward(self, x_cat, x_cont): z = torch.cat([e(x_cat[:, i]) for i, e in enumerate(self.embs)], dim=1) return self.head(torch.cat([z, self.bn(x_cont)], dim=1))

class SeqModel(nn.Module): def __init__(self, in_dim, hidden=64, out_dim=1, p=0.2): super().__init__() self.lstm = nn.LSTM(in_dim, hidden, num_layers=2, batch_first=True, dropout=p) self.head = nn.Linear(hidden, out_dim) def forward(self, x): # x: (batch, seq_len, in_dim) out, _ = self.lstm(x) return self.head(out[:, -1, :]) # last timestep

▲ Canonical training loop

def train_epoch(model, dl, loss_fn, opt, device): model.train() tot, n = 0.0, 0 for xb, yb in dl: xb, yb = xb.to(device), yb.to(device) opt.zero_grad(set_to_none=True) # faster than zero_grad() pred = model(xb).squeeze(-1) loss = loss_fn(pred, yb) loss.backward() # Optional: clip gradient # torch.nn.utils.clip_grad_norm_(model.parameters(), max_norm=1.0) opt.step() tot += loss.item() * xb.size(0); n += xb.size(0) return tot / n @torch.no_grad() def eval_epoch(model, dl, loss_fn, device): model.eval() # disables Dropout, BN uses running stats tot, n = 0.0, 0 for xb, yb in dl: xb, yb = xb.to(device), yb.to(device) pred = model(xb).squeeze(-1) tot += loss_fn(pred, yb).item() * xb.size(0); n += xb.size(0) return tot / n
Things that MUST be in every loop: opt.zero_grad() before backward; model.train()/model.eval() toggles; with torch.no_grad() on eval; move tensors to device.

from torch.optim.lr_scheduler import ( ReduceLROnPlateau, CosineAnnealingLR, OneCycleLR) # Reduce on plateau (simple, robust): sched = ReduceLROnPlateau(opt, mode="min", factor=0.5, patience=5) sched.step(va_loss) # call AFTER eval # Cosine annealing (transformer-style): sched = CosineAnnealingLR(opt, T_max=epochs, eta_min=1e-6) sched.step() # call once per epoch # OneCycle (super-fast convergence; Smith 2018): sched = OneCycleLR(opt, max_lr=1e-2, total_steps=epochs * len(dl_tr)) # Call after EVERY batch.step()
Mixed precision (Ampere+ GPU)
# PyTorch 2.4+ canonical API (torch.cuda.amp.* deprecated): DEV = device.type # "cuda" or "cpu" — works on either scaler = torch.amp.GradScaler(DEV, enabled=(DEV=="cuda")) for xb, yb in dl_tr: opt.zero_grad(set_to_none=True) with torch.autocast(device_type=DEV, dtype=torch.float16, enabled=(DEV=="cuda")): pred = model(xb) loss = loss_fn(pred, yb) scaler.scale(loss).backward() scaler.step(opt) scaler.update() # ~2× speed, ~50% memory; use bf16 on H100/A100 for numerical stability

▲ Hyperparameter tuning

Optuna with PyTorch
import optuna def objective(trial): hidden = trial.suggest_categorical("h", [32, 64, 128, 256]) n_lay = trial.suggest_int("n_lay", 1, 4) p = trial.suggest_float("p", 0.0, 0.5) lr = trial.suggest_float("lr", 1e-4, 1e-2, log=True) wd = trial.suggest_float("wd", 1e-6, 1e-2, log=True) model = MLP(in_dim, hidden=tuple([hidden]*n_lay), p=p).to(device) opt = torch.optim.AdamW(model.parameters(), lr=lr, weight_decay=wd) best_va = train_to_convergence(model, opt, dl_tr, dl_va) return best_va study = optuna.create_study(direction="minimize", pruner=optuna.pruners.MedianPruner()) study.optimize(objective, n_trials=50, n_jobs=1) print(study.best_params)
Tuning priority order
  1. Learning rate (log scale 1e-4 to 1e-2)
  2. Batch size (power of 2; affects LR scaling)
  3. Architecture (hidden width, depth)
  4. Dropout / weight decay (regularisation strength)
  5. Scheduler (ReduceLROnPlateau vs OneCycle)
  6. Optimizer (Adam vs AdamW vs SGD)
LR finder (smith 2017)
# Quick LR sweep — increase LR exponentially each batch, find where loss explodes lrs, losses = [], [] opt = torch.optim.SGD(model.parameters(), lr=1e-7) for i, (xb, yb) in enumerate(dl_tr): lr = 1e-7 * (10 ** (i / 30)) for g in opt.param_groups: g["lr"] = lr opt.zero_grad(); l = loss_fn(model(xb.to(device)).squeeze(-1), yb.to(device)) l.backward(); opt.step() lrs.append(lr); losses.append(l.item()) if l.item() > 4 * min(losses): break # Plot lrs vs losses; pick LR one decade below the explosion point

▲ Top traps (6-month memory)

  • Forgetting opt.zero_grad() — gradients accumulate; training silently breaks.
  • Double sigmoid/softmax — sigmoid + BCEWithLogitsLoss = double-applied. Use raw logits + BCEWithLogitsLoss.
  • Scaler fit on all data — leakage. Fit on TRAIN only, transform val/test.
  • Peeking at test during tuning — every peek poisons the held-out estimate. Use val for tuning.
  • Missing model.eval() — Dropout still active + BN uses batch stats → noisy/wrong predictions.
  • Reporting in scaled units — inverse-transform predictions before metric calls.
  • Judging imbalanced classifiers by accuracy — 99% acc on 1% positives = "predict 0 always".
  • Dtype mismatch — CrossEntropyLoss wants long; BCEWithLogitsLoss wants float. Mismatch silently fails or NaNs.
  • No deterministic seed for cuDNN — set torch.backends.cudnn.deterministic = True for full reproducibility (small speed cost).
  • Not overfit-testing a single batch — first sanity check; if you can't overfit one batch, the model/loss is wrong.
